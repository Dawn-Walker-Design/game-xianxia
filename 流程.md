## 一、PLY → GLB 转换流程

### 1.1 原始数据

- 文件: `1ba62e053253fc31e60c97ebfd436aee.ply`
- 大小: 39 MB (binary little-endian)
- 顶点: 781,985
- 面: 1,502,744
- 颜色: RGBA uchar (22~255)，原始颜色偏灰蓝色系
- 尺寸: X=-11.7~10.6 (22m), Y=-9.1~12.5 (22m), Z=-3.4~8.2 (12m)
- 这是一个完整的 3D 扫描世界场景，不是单个模型

### 1.2 居中并落地

```python
import trimesh
m = trimesh.load('xxx.ply', force='mesh')
b = m.bounds
m.vertices[:, 0] -= (b[0][0] + b[1][0]) / 2  # X 居中
m.vertices[:, 1] -= b[0][1]                    # 落地 (Y≥0)
m.vertices[:, 2] -= (b[0][2] + b[1][2]) / 2  # Z 居中
```

### 1.3 导出 GLB

```python
m.export('world_full.glb', file_type='glb')
```

导出后 29 MB，保持原始顶点色和面数。

## 二、踩坑记录

### 坑1：trimesh 减面必丢颜色 ⚠️⚠️⚠️

**现象**：调用 `m.simplify_quadric_decimation(face_count=50000)` 后，所有顶点色变成 (102,102,102) 纯灰。

**原因**：`fast_simplification` 库在减面时丢弃了原始顶点色，用默认灰色填充。

**尝试过的方案**：
- `percent` 参数 → 同样丢失
- `face_count` 参数 → 同样丢失
- 逐级减面 → 每一步都丢，100% 丢失率
- vertex clustering (numpy 手工实现) → 颜色能保留，但几何细节被磨平

**当前结论**：trimesh 减面 = 必然丢色。如果想要颜色，要么不减面，要么用其他工具。

### 坑2：vertex clustering 破坏几何

**做法**：把空间按 0.15m 网格划分，同格顶点合并

**结果**：78万顶点 → 7万顶点，但：
- 很多面变成退化面（三顶点在同格内）
- 剩余面的边长全是 <0.5m
- 山体细节完全消失，看起来像低多边形 blob

**结论**：vertex clustering 不适合地形类模型，只适合密集点云。

### 坑3：JS 端循环处理顶点色会卡死

**尝试**：加载 GLB 后，在 Three.js 中对 78 万顶点做颜色增强

**结果**：
- `src.getX(i)` 调用 78 万次 → 浏览器卡死数十秒
- 即使是 `Float32Array` 直接操作也很慢

**结论**：所有颜色处理必须在 Python 端完成。

### 坑4：世界模型尺寸理解

**关键认知**：这个 PLY 不是"一个建筑"或"一个道具"，而是**整个 3D 世界**——包含地形、山体、水面、建筑等所有内容。

- 原始尺寸 22m × 22m × 12m
- scale=5 后 110m × 110m × 60m
- 建筑（大门/山门）需要放在世界边缘外侧，作为入口

### 坑5：150万面 vs M1 8GB 的性能

**当前状态**：直接加载 29MB / 150万面的 world_full.glb

**风险**：
- M1 8GB 统一内存，GPU 和 CPU 共享
- 150 万面的 vertexColors 在 GPU 端可能占用大量内存
- 预期帧率：5-15 fps（取决于视口大小和相机距离）

**优化方向**（还没做）：
- 用 MeshoptCompression 压缩 GLB
- 用 Three.js SimplifyModifier 在加载后减面
- 接受低帧率，因为是探索类游戏而非动作类

### 坑6：颜色增强策略

原始 PLY 颜色分布：62%蓝色系 + 47%灰白色 + <1%绿色

**尝试过的方案**：
1. JS 端逐顶点增强 → 太慢
2. Python 端统一饱和度增强 → 效果一般（原始颜色太偏蓝）
3. Python 端按高度分层着色 → 效果最好

**分层着色方案**（效果最好）：
```python
水(Y<1.5m): 深蓝 (25, 80, 190)
地面(1.5-5m): 绿色 (50, 140, 40)
低山(5-10m): 黄绿 (110, 150, 60)
中山(10-16m): 棕色 (160, 130, 90)
高山(16m+): 灰白 (235, 230, 220)
```

但由于减面丢色的坑，这个方案需要配合不减面的 GLB 使用。

## 四、Three.js 渲染配置

### 4.1 模型加载

```javascript
const gltfLoader = new GLTFLoader();
gltfLoader.load('/models/world_full.glb', (gltf) => {
  worldModel = gltf.scene;
  worldModel.scale.setScalar(5);
  worldModel.position.set(0, 5, 0);
  worldModel.traverse(node => {
    if (node.isMesh) {
      node.material = new THREE.MeshLambertMaterial({
        vertexColors: true,
        side: THREE.DoubleSide,
      });
    }
  });
  scene.add(worldModel);
});
```

### 4.2 灯光方案

```
太阳光: DirectionalLight(白色, intensity=10)  — 主光源
补光: DirectionalLight(蓝白, intensity=4)     — 减少阴影
环境光: AmbientLight(浅蓝, intensity=6)        — 避免全黑
半球光: HemisphereLight(天蓝/绿, intensity=3)  — 天地颜色渐变
```

### 4.3 相机

```
PerspectiveCamera(FOV=70, near=0.5, far=1000)
初始位置: (0, 50, 100)
注视点: (0, 20, 0)
```
