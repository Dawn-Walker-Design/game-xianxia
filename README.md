# 天庭仙境 · 仙侠 3D 探索游戏

> **Xianxia Celestial Realm — 3D Exploration Web Game**

---

## 项目概述

基于 **Three.js + Vite** 的仙侠天庭主题 3D 网页探索游戏。玩家出生在云端白玉平台上，探索由 AI 生成的 3D 建筑模型组成的磅礴天庭场景。

---

## 快速启动

```bash
cd /Users/hyf/xianxia-explorer
npx vite --port 3000
```

浏览器打开 `http://localhost:3000`，点击画面锁定鼠标后即可探索。

## 场景构成

### 核心平台
- **白玉天阶**：半径 80m 的白玉圆台，带金色边饰和装饰环纹
- **中央金镶纹**：平台正中的金色圆形图案（出生点）

### 建筑群（6 个 GLB 模型）

| 建筑 | 模型文件 | 目标尺寸 | 位置 |
|---|---|---|---|
| 凌霄宝塔 | building_pagoda.glb | 46m宽 × 59m高 | 中轴线远后方 (0, -70) |
| 通天灵树 | building_spirittree.glb | 41m宽 × 37m高 | 中轴线后方 (0, -35) |
| 太虚宝殿 | building_temple.glb | 47m宽 × 38m高 | 左前方 (-55, -15) |
| 天门·右 | building_shanmen.glb | 48m宽 × 50m高 | 右前方 (55, -15) |
| 玉虚仙桥 | building_bridge.glb | 45m宽 × 33m高 | 左后方 (-40, -55) |
| 内天门 | gate.glb | 50m宽 × 40m高 | 正面入口 (0, 25) |

所有建筑自动应用天庭配色：**白玉基座 → 朱红柱身 → 金色琉璃瓦**（运行时按高度分层着色）

### 远景元素
- **金色天门**：代码构建的巨型门坊，位于平台后方
- **6 组宫殿剪影**：金顶建筑轮廓分布在地平线上
- **55 座远峰**：围绕地平线的渐变山峰环

### 云海系统
- **4 层体积云海**：平台下方共 230 个浮动云团
- **8 个浮空云台**：散落在平台外围，飞行模式可到达
- **35 个云碎片粒子**：平台高度漂浮

### 粒子系统
- **200 个金色光点**：天庭灵气光华（叠加混合，缓慢漂浮）
- **80 片桃花瓣**：粉色落花（法线混合，飘落循环）
- **35 个白云碎片**：雾状漂浮物

### 神光与天气
- **主方向光**：柔和金色 `#fff8e8`，模拟天庭神光
- **8 道丁达尔光束**：从天空射下的半透明光柱，缓慢旋转脉冲
- **体积雾**：`FogExp2`，远处景物渐入云雾
- **渐变天空球**：半径 480m，顶深蓝 → 中天青 → 地平雾白金

### 可交互道具
- **丹炉**：位于 (-18, 0.05, 10)，靠近显示"按 E 查看"，炉口有火光脉冲
- **仙剑**：位于 (18, 0.05, -12)，靠近显示"按 E 收集"，收集后半透明

### 探索进度 UI
- 顶部显示 `探 索 进 度 ● ● 0 / 2`
- 每发现一个道具，对应圆点亮起金色
- 全部收集后显示 `✨ 探索完成 ✨`

## 模型处理流程

### 原始文件
```
/ (根目录)
├── *.ply          # 原始 PLY 世界模型（当前未使用）
├── *.glb          # AI 生成的原始高面数 GLB 模型（60-80MB）
└── public/models/
    ├── building_*.glb   # 处理后的建筑模型（~1-2MB，45K面）
    ├── gate.glb         # 处理后的山门模型（1.1MB，45K面）
    ├── world_full.glb   # 处理后的世界模型（29MB，当前未加载）
    └── world_area2.glb  # 处理后的第二世界（16MB，当前未加载）
```

### 处理步骤
1. **AI 生成** → 原始高面数 GLB（~1M 面，60-80MB）
2. **Python 减面** (`process_new_models.py`) → 用 `fast_simplification` 减至 ~45K 面
3. **居中落地** → 将模型中心移至原点，底部着地
4. **JS 端着色** → 运行时按高度应用天庭配色，写入 `vertexColors`
5. **渲染** → `MeshPhongMaterial(vertexColors: true, side: DoubleSide)`

### 模型尺寸说明
- AI 生成的原模型尺寸差异大：gate.glb 原始 10m，其他模型原始仅 0.7-1.0m
- 代码中用 scale 统一放大到 40-60m 的磅礴比例
- gate.glb（山门）原始就大，用 scale=5.0 放大到 50m

## 技术架构

```
index.html          # 单文件应用，内嵌 HTML + CSS + JS
├── importmap       # Three.js 通过 importmap 引入
├── GLTFLoader      # 加载 /models/ 下的 GLB 模型
├── vertexColors    # 运行时生成天庭配色顶点色
├── FogExp2         # 体积雾
├── Raycaster       # 交互检测（道具靠近提示）
└── requestAnimationFrame  # 60fps 游戏循环
```

### 技术栈
- **Three.js** v0.184（ES 模块，importmap）
- **Vite** v6.4（开发服务器）
- **Python** + trimesh + fast_simplification（模型预处理）
- **无其他框架依赖**

### 渲染配置
- `ACESFilmicToneMapping`，曝光 1.15
- 无阴影（`shadowMap.enabled = false`，适配 M1 GPU）
- 像素比限制 `min(devicePixelRatio, 2)`

## 性能说明

- **M1 8GB**：6 个 45K 面模型 + 云层 + 粒子 ≈ 稳定 30-60fps
- 不使用世界 PLY 模型（150万面太重）
- 未启用阴影以节省 GPU 资源
